# 首次分享执行清单

这个清单用于把 Mama Save JP 第一次发到小红书、朋友圈或小范围群聊前做人工检查。当前阶段只验证用户是否愿意打开、收藏、回访和转发，不做表单、登录、付费群、抽奖或复杂追踪。

## 适用场景

- 准备把站点发给 5-10 个在日华人宝妈/宝爸。
- 准备从 `docs/growth-copy-bank.md` 里挑 1 条文案发布。
- 准备把某个页面发到微信群、LINE 群或朋友圈。
- 收到亲友反馈后，想判断下一步是改内容、改入口，还是继续收集样本。

## 发布前 10 分钟检查

每次只选一个主入口，不要同时推首页、优惠、日历和多篇攻略。

1. 选定目标用户：孕期、0-6 个月、7-18 个月、19-36 个月，或刚来日本的新手家庭。
2. 选定一个入口：`/`、`/deals`、`/calendar`，或 1 篇具体攻略。
3. 打开对应页面，确认标题、首屏和主要卡片能解释“适合谁”和“怎么判断”。
4. 如果文案提到优惠，只引用站内 `dataStatus: verified` 且 `availabilityStatus: active` 的内容。
5. 如果不确定优惠是否仍有效，就改发攻略或日历解释，不要发“本周值得买”。
6. 删掉夸张词：必买、稳赚、闭眼入、全网最低、一定返、错过就亏。
7. 在文案末尾只放一个链接，避免用户不知道先点哪里。
8. 准备 1 个追问问题，发布后只追问最关键的反馈。

## 反馈处理

- 如果 3 人以上说看不懂站点用途，优先改首页第一屏或入口文案。
- 如果 3 人以上担心优惠过期，优先复查 `/deals` 和来源说明。
- 如果 3 人以上只收藏攻略、不看优惠，下一轮优先补攻略厚度和内链。
- 如果 3 人以上愿意转发某一页，保留原入口，不急着改版。
- 如果反馈只是单个主题缺内容，记录到后续攻略或优惠数据任务，不做大范围重构。

## 不做事项

- 不用未经确认的截图或价格吸引点击。
- 不承诺一定省钱或固定返现。
- 不把抽选活动写成确定优惠。
- 不在早期收集手机号、宝宝生日、家庭收入或详细消费记录。
- 不因为单条反馈马上改导航、设计或核心结构。
